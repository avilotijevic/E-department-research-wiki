# Hábrók — high-performance computing (HPC) cluster

Hábrók is the University of Groningen’s high-performance computing (HPC) cluster.

## Specifications

    84 computers with 128 cores and 512 GB memory. These systems are specially tailored for data-intensive tasks;
    48 computers with 128 cores, 512 GB memory and a super fast, low latency interconnect. These systems are suitable for large-scale processing of multiple computers;
    4 nodes with 80 cores and 4 TB of memory for memory-intensive tasks;
    3 nodes with 128 cores, 1.5 TB of memory and each 8 Nvidia RTX Pro 6000 GPU's, amongst other things suitable for the most new AI-workloads;
    6 nodes with 64 cores, 512 GB memory and 4 Nvidia A100 cards with 40 GB memory each. These can, for example, accelerate machine learning;
    19 nodes with 36 cores, 768 GB memory and 2 Nvidia V100 cards with 32 GB memory each, amongst other things suitable for AI-applications.
    3PB shared storage for data to be processed on or generated by the cluster

## Getting an account

To use **Hábrók**,you first need to request access via the university’s support system.

#### Step 1: Submit an access request
Go to the following website:  
https://rug.topdesk.net/tas/public/ssp/

Then navigate through:

Research and Innovation Support  
→ Computing and Research Support Facilities  
→ Computing (Hábrók, Merlin)  
→ Request Hábrók account  

#### Step 2: Provide required information
After submitting your request, you will receive a follow-up message asking you to provide additional details, typically including:

- Your RUG username  
- Your research group or supervisor  
- The project you will use Hábrók for  
- A short description of your intended computations  

#### Step 3: Wait for approval
Once your request is approved, you will receive:

- Access credentials  
- Instructions for logging in (via SSH)  
- Information on how to start using the cluster  

### Connecting to Hábrók

After your account has been approved, you can connect to **Hábrók**.

---

#### Step 2: Install a terminal / SSH client

You need a way to connect to the cluster via SSH.

**Windows**
- Download: https://mobaxterm.mobatek.net/  
- The free Home Edition is sufficient  

**macOS / Linux**
- You can use the built-in terminal (`ssh`, `scp`)  
- Optionally, use a graphical SFTP client (e.g., FileZilla) to transfer files more easily  

---

#### Step 3: Connect via SSH

Use one of the **login nodes**:

- `login1.hb.hpc.rug.nl`  
- `login2.hb.hpc.rug.nl`  

These are the main entry points to the system.

**Example (terminal):**

    ssh p123456@login1.hb.hpc.rug.nl

- Replace `p123456` with your own RUG username  
- Make sure the **“p” is lowercase**  

You will then:
1. Enter your password  
2. Complete the two-factor authentication (e.g., Google verification)  

> **Note:** When typing your password in the terminal, nothing will appear on screen. This is normal.

---

#### Types of nodes (important)

- **Login nodes (`login1`, `login2`)**
  - Used to log in, manage files, and submit jobs  
  - **Do not run heavy computations here**

- **Interactive nodes (`interactive1`, `interactive2`)**
  - Used for short tests and debugging  
  - Not intended for long or intensive jobs  

- **GPU nodes (`gpu1`, `gpu2`)**
  - Used for GPU-based computations (e.g., machine learning)  
  - Typically accessed via the job scheduler  

---

#### Important note

All heavy computations should be run via the **job scheduler (SLURM)**, not directly on login or interactive nodes.
